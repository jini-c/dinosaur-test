프로젝트 기획과 기술 요구사항

성격 테스트 웹사이트의 특징

  • 참여 기반 콘텐츠
    단순 정보 전달이 아닌, 사용자의 선택과 반응을 유도하는 상호작용 중심 콘텐츠
  • 개인화된 결과 제공
    사용자마다 다른 결과를 제공하여 재미와 몰입도, 만족감을 높임
  • 간결한 참여 흐름
    총 10개의 직관적인 문항으로 누구나 쉽게 참여 가능

웹 기술 요구사항

  • 설치 없이 접근 가능
    별도의 앱 설치 없이 웹 브라우저만으로 바로 이용 가능
  • 자연스러운 전환
    페이지 새로고침 없이 부드럽게 다음 화면으로 전환
  • 모바일 최적화
    다양한 기기에서 화면이 무너지지 않고 사용 편의성 유지
  • 빠른 결과 계산
    서버 없이 클라이언트에서 즉시 계산해 결과 제공

🔧 개발 과정의 주요 도전과 해결

❌문제 1: 페이지 전환이 안 되는 오류

문제: showPage() 호출했는데 화면이 바뀌지 않음

원인: DOM 업데이트 타이밍과 함수 호출 순서

해결: 브라우저 개발자 도구로 디버깅, 올바른 실행 순서 파악

🖼️문제 2: 결과 이미지 로딩 실패

문제: 공룡 이미지가 표시되지 않음

원인: 잘못된 파일 경로 설정

해결: 상대 경로 개념 이해하고 정확한 경로 설정
